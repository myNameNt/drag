  1. 1.分布式，每个开发者都是一个服务器
  2. 2.通过github进行多人协作
  3. 3.git分为工作区 暂存区 版本库
  4. 4. 使用个git status 查看是否有文件改动 git add +文件名 提交单个的文件到暂存区 git add . 提交所有变动的文件
  5. 5.使用git commit -m ‘你的注释’ 将暂存区的文件内容提交到版本库（ps：这里的版本库不是github远程版本库）
  6. 6.使用git commit -a -m ‘你的注释’ 是 add操作和commit操作的简写形式
  7. 7.使用git diff 查看工作区和暂存区文件的区别
  8. 8.使用git diff --cached 查看暂存区和版本去文件的区别
  9. 9.使用git reset HEAD 将刚才git add 的添加到暂存区的内容撤回到以前的状态
  10. 10.使用git checkout --文件名 可以将让工作区该文件的内容还原成本地版本库该文件的内容
  11. 11.使用git commit -m '我重新修改了的提交' --amend 可以合并上次的提交，（使用场景：当上一次的提交没有全部提交我需要修改的内容时，但是又不想有两次提交记录）
  12. 12.使用git rm +文件名 可以删除暂存区的文件（ps：要删除暂存区的文件你必须先删除工作区的文件）
  13. 13.使用git rm -f +文件名 可以同时删除工作区和暂存区的该文件
  14. 14.使用git rm --cached +文件名 可以删除暂存区的该文件，但是工作去依然保留
  15. 15.使用git checkout +git log 里面commit 的id 要恢复的文件名 可以回到这个commit的版本(ps:还原指定文件)
  16. 16.使用git reset --hard +commit 的id 恢复到指定的版本 （ps：所有文件）
  17. 17.使用git reset --hard HEAD^ 可以回到上一个版本（HEAD就像是一个指针，使用这个命令表示将指针移到最近的过去的一个版本）
  18. 18.使用git reset --hard HEAD~后面接一个数字 可以回到过去的指定的第几个版本
  19. 19.使用git remote 查看远程分支名 git remote -v 查看远程分支名和路径
  20. 19.使用git reflog 查看所有commit操作和 git reset操作记录。可以使用对应的id 然后使用git reset --hard +id 回到那个版本
  21. 20.使用git fetch 更新同步更新远程master到本地,由于多人合作时可能发生冲突（两个人或者多人修改了同一个地方的代码）发生冲突时使用 git diff master origin/master 查看本地master和远程master的区别 然后使用 git merge origin/master 合并 然后商量如何修改解决冲突 再做提交
  22. 21.使用git pull origin（若默认可不写） 自动合并 若有冲突使用远程master的代码替换本地的代码
  23. 22.使用git branch 查看当前项目所有本地分支 使用git branch +自定义分支名 创建一个新的分支
  24. 23.使用git checkout +分支名 切换到该分支
  25. 24.使用git checkout -b +自定义分支名 创建新的分支并切换到该分支（上面两步骤的简写）
  26. 25.使用git merge +分支名 将分支内容合并到主分支
  27. 26.使用git branch --merged 查看与主分支已经合并的分支 git branch --no-merged 查看未能与主分支合并的分支
  28. 27.使用git branch -d +分支名 可以删除本地该分支
